<p class="section-kicker not-content">Why it exists</p>

<div class="feature-grid not-content">
  <article class="feature" data-reveal>
    <span class="twig" aria-hidden="true">❧</span>
    <h3>Search by similarity</h3>
    <p>Hand nidus a query vector and get the closest records back, ranked with scores. Exact by default — 100% recall at the scale it targets — by cosine, dot, or Euclidean. Reach for an approximate index (HNSW or IVF) or int8/binary quantization when you want more speed.</p>
  </article>
  <article class="feature" data-reveal style="--reveal-delay:0.08s">
    <span class="twig" aria-hidden="true">❧</span>
    <h3>Filter while you search</h3>
    <p>Store typed metadata — Str, Int, Bool, List — beside every vector and narrow the results before they score: equality, ranges, globs, sets, negation. Group records into collections that share one embedding space and query them in a single call.</p>
  </article>
  <article class="feature" data-reveal style="--reveal-delay:0.04s">
    <span class="twig" aria-hidden="true">❧</span>
    <h3>One directory, crash-safe</h3>
    <p>A whole store is a single append-only directory, fsync'd per batch and CRC-checked. A crash loses at most the in-flight write, and a torn tail repairs itself the moment you reopen — nothing to back up but a folder.</p>
  </article>
  <article class="feature" data-reveal style="--reveal-delay:0.12s">
    <span class="twig" aria-hidden="true">❧</span>
    <h3>From a folder to the cloud</h3>
    <p>Start against a local directory, then move the same store to S3, GCS, or Redis/Valkey by changing one URL — no migration, no rewrite. Begin on a laptop and grow into the cloud when you need to.</p>
  </article>
</div>
